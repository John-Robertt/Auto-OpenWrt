# Auto-OpenWrt 产品策划 v1

## 产品定位

Auto-OpenWrt 是一个面向个人玩家的自动化 OpenWrt 固件构建工具。

它的核心职责是让用户通过声明式配置完成源码更新、插件和 feeds 接入、固件构建、失败诊断与自动修复协同，尽量减少用户对宿主系统环境、OpenWrt 构建细节和重复操作的关注。

## 核心目标

- 让用户在 macOS、Linux、WSL 上获得尽量一致的固件构建体验。
- 让用户用配置描述 OpenWrt target、源码、feeds、插件、更新策略和构建选项。
- 允许一个 project root 管理多个配置文件，并让每个配置文件拥有独立 workspace，以支持不同固件版本并行维护。
- 支持全部 OpenWrt target 的配置、源码更新和构建流程。
- 支持自定义 feeds 和插件源码，覆盖 OpenClash、Turbo ACC 这类常见玩家需求。
- 每次构建前自动检查系统环境、Docker、AI 工具、运行权限、磁盘空间、网络、project root 和 workspace 状态。
- 默认跟随最新源码更新，并在成功构建后记录可追踪版本信息。
- 构建失败后自动整理上下文，并调用外部 AI CLI 尝试修复源码与重新构建。
- AI 修复后的构建成功时，自动把修复差异采纳为可审计补丁，后续同配置和同 build 构建可复用。
- 输出完整固件，并保留日志、配置解析结果、源码版本和构建记录。

## 产品职责

### 配置驱动构建

用户只维护构建配置，工具负责按配置更新源码、选择 target、启用插件并发起构建。

一个 project root 可以保存多个配置文件；每个配置文件对应一个 workspace，并隔离 run record、运行工作树、产物、success lock 和 adopted patches。

### Target 管理

支持用户通过 build 选择任意 OpenWrt target、subtarget 和 device profile，并在构建流程中保留对应配置和产物信息。

### 健康检查

每次构建前检查系统环境依赖、Docker 可用性、AI 工具状态、CLI 运行权限、project root 权限、workspace 权限、磁盘空间、网络能力和配置完整性。

关键检查失败时，产品需要阻断构建并给出明确、可执行的修复建议。

### 源码更新管理

支持 OpenWrt、feeds、插件源码跟随指定分支最新版本；成功构建后记录本次成功源码状态。

### 自定义源管理

允许用户声明多个 feeds、多个插件仓库、插件路径、启用状态和插件分类。

### 插件风险分层

产品需要区分普通应用/LuCI 插件与内核模块/补丁型插件，并在提示、日志和失败诊断中明确体现风险。

### 失败处理闭环

构建失败后自动收集失败包、日志片段、配置、源码版本和修改范围，调用用户配置的 AI CLI 进行修复尝试。

修复后构建成功时，产品自动归档 adopted patch，并让该 build 后续构建复用成功修复；修复失败时只保留现场和历史，不更新成功状态。

### 自动重试控制

AI 修复后最多自动重试 5 次，超过后停止并保留现场。

### 产物管理

成功构建后保留固件、日志、解析后配置、成功版本记录；失败时保留失败现场和诊断信息。

### 用户体验简化

用户不需要手动进入 OpenWrt 源码目录执行复杂命令；常见流程应由单条构建命令完成。

## 验收标准

- 用户可以通过配置选择任意 OpenWrt target、subtarget 和 device profile 并发起完整固件构建。
- 每次构建前自动完成健康检查，关键环境、权限、Docker 或 AI 工具问题会阻断构建并输出修复建议。
- 用户可以添加自定义 feeds 和插件，并在构建配置中启用。
- 工具可以更新 OpenWrt、feeds、插件源码，并记录成功构建时的版本信息。
- 构建失败时，工具可以生成完整诊断上下文并触发 AI 修复流程。
- AI 修复流程最多重试 5 次，失败后保留现场而不是无限循环。
- AI 修复成功后自动生成 adopted patch，并能从成功产物追溯到对应修复差异。
- 成功产物包含固件、日志、解析后配置、健康检查报告和源码版本记录。
- 产品能明确提示 OpenClash 与 Turbo ACC 的不同风险等级。

## 默认假设

- 用户接受纯配置 CLI 作为 v1 主交互方式。
- 用户希望尽量自动化，但仍接受失败后查看日志和现场。
- 用户更重视跟随最新源码和插件版本，而不是每次固定旧版本构建。
- 用户接受 AI 修复成功后自动采纳为可审计补丁。
- 用户本机或运行环境已具备基础容器构建条件。
- 外部 AI CLI 由用户自行安装、登录和授权，Auto-OpenWrt 负责调用与上下文编排。
